2. The Architectural Split: Semantic Layer vs. Context LayerTo build bulletproof enterprise AI agents, architects must decouple the calculation of data from the governance of behavior. While often conflated, the Semantic Layer and the Context Layer solve two entirely different pieces of the trust puzzle.  The Semantic Layer: The Mathematical CoreThe Semantic Layer translates physical database structures (tables, primary/foreign keys, columns) into single, declarative logical representations of business terminology.  Core Artifacts: Stores metric formulas (e.g., gross_margin), dimension hierarchies (e.g., Country -> State -> City), and calculated measures.  Determinism Engine: It answers the question, "What does this metric mathematically mean, and how is it compiled?" It forces an LLM to request data through a locked, strictly audited semantic definition rather than generating raw, unverified SQL joins.  The Context Layer: The Operational ShellThe Context Layer wraps around the Semantic Layer. It maps how data fragments across completely independent software ecosystems and determines the rules under which an agent may interact with that data.  Core Artifacts: Cross-system ontologies, system lineage, access control policies, data quality signals, and historical decision memory.  Behavioral Guardrail: It answers the questions, "When, how, and under what constraints can an AI agent use this data? Is this data currently trusted? Who owns it?"
